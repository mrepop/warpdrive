# Contributing to WarpDrive

Thank you for your interest in contributing to WarpDrive!

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/warpdrive.git`
3. Create a feature branch: `git checkout -b feature/my-feature`
4. Make your changes
5. Commit your changes with clear messages
6. Push to your fork: `git push origin feature/my-feature`
7. Create a Pull Request

## Development Setup

1. Ensure you have Xcode 26.0+ installed
2. Open the project in Xcode
3. Build and run to verify everything works

## Code Style

- Follow Swift naming conventions
- Use SwiftUI for UI components
- Write clear, self-documenting code
- Add comments for complex logic
- Keep functions small and focused

## Testing

- Write unit tests for new features
- Ensure all tests pass before submitting PR
- Add UI tests for critical user flows

## Pull Request Guidelines

- Provide a clear description of the changes
- Reference any related issues
- Include screenshots for UI changes
- Ensure CI passes

## Questions?

Open an issue for discussion or reach out to the maintainers.
